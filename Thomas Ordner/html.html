Welche Funktion hat ein Browser?

  • Interpretation von HTML: Der Browser liest HTML-Code und interpretiert ihn, um die Struktur und das layout einer Webseite zu verstehen.

  • Darstellung von HTML: Der Browser wandelt HTML-Code in ein visuelles Layout um

  • CSS-Unterstützung: Der Browser unterstützt CSS, um das Aussehen und Layout von HTML-Webseiten zu verbessern.

  • JavaScript-Unterstützung: Der Browser unterstützt JavaScript, um interaktive Funktionen auf HTMl-Webseiten zu ermöglichen.

  • Entwicklertools: Der Browser bietet diese Tools an, um HTML-, CSS- und JavaScript-Code zu inspizieren und zu debuggen.

Wie geht ein Browser mit einer HTML Seite um?

  • Lädt HTML-Datei vom Webserver herunter, nachdem der Benutzer eine URL eingegeben hat oder auf einen Link geklickt hat.

  • Analysiert HTML-Code, um die Struktur und das Layout der Webseite zu verstehen.

  • Lädt Ressourcen wie Bilder, CSS-Dateien und JavaScript-Dateien, um das visuelle Layout der Webseite zu erstellen.

  • Wendet CSS auf HTML-Elemente an, um das visuelle Layout und Aussehen der Webseite zu verbessern.

  • Führt JavaScript-Code aus, um interaktive Funktionen auf der Webseite zu ermöglichen.

  • Rendert die Webseite, indem er HTML-, CSS-, JavaScript-Code verarbeitet und ein visuelles Layout anzeigt.

  • Benutzer kann mit der Webseite interagieren, indem er auf Links klickt, Formulare ausfüllt oder andere Aktionen ausführt.

Was passiert beim Parsen?

  • HTML-Code wird in seine Bestandteile zerlegt.

  • HTML-Code wird in einer Baumstruktur namens DOM-Struktur (Document Objekt Model) organisiert.

  • Durch das Parsen kann der Browser den Inhalt der Webseite verstehen und entsprechend anzeigen.

  • HTML-Code wird in kleinere Einheiten, z.B. in Tags und Attribute aufgeteilt und dann in eine Baumstruktur eingeordnet, die die Hierarchie und Beziehungen der verschiedenen HTML-Elemente darstellt.

HTML Semantic

  • "semantic elements = elements with a meaning"
  • Semantische Elemente beschreibt seine Bedeutung für den Browser und für den Benutzer
  • non-semantic elements: <div> und <span> sagen nichts über deren Inhalt aus
  • semantic elements: <form>, <table>, <article> beschreiben klar deren Inhalt
